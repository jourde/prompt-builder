I am going to give you a list containing prompts collected from a group of educational support coordinators in European Schools. They were asked to describe their professional needs so that we can adapt a prompt-building tool to their work.

Your task is to consolidate the raw responses into a **clean, standardised, option-ready list** that can later be used to create selectable options in the prompt creation interface.

At this stage, you are **not** designing the interface.  
You are **not** deciding menus, control types, field structure, or layout.  
You are only preparing the content base from which interface options may later be derived.

The purpose of this task is to:
- identify recurring meanings in the raw responses
- merge only genuinely equivalent ideas
- preserve distinctions that may matter later as separate options
- produce a structured list suitable for later interface design

Use these four headings only:

# PERSONA / ROLE
# OBJECTIVE
# AUDIENCE
# BOUNDARIES & STYLE

For each heading, follow this process:

## STEP 1 — MERGE
Identify responses that express the same idea, even if worded differently.  
Merge them into one item only when the meanings are clearly equivalent.

## STEP 2 — STANDARDISE
Write each merged item as a short, self-contained phrase, not a full sentence.  
Use consistent grammatical form within each heading.  
Write items so they could later serve as candidate interface options.

## STEP 3 — ORGANISE
Sort items from most to least frequently mentioned.  
Add frequency counts in brackets, for example `(×4)`.  
Flag items mentioned only once with `[unique]`.  
Flag items that remain ambiguous or semantically unclear with `[to clarify]`.

Important rules:
- Do not invent items.
- Do not discard any response silently.
- Preserve respondent terminology where possible.
- Work from meaning, not wording.
- Produce all output in English.
- Keep PERSONA / ROLE, OBJECTIVE, AUDIENCE, and BOUNDARIES & STYLE separate.
- Do not design the interface yet.
- Do not recommend fields, menus, or control types.
- Do not collapse distinct ideas if they may later need to become separate options.
- Aim for a consolidation that is usable as a later option pool.

Required output:

# PERSONA / ROLE
[numbered consolidated list]

# OBJECTIVE
[numbered consolidated list]

# AUDIENCE
[numbered consolidated list]

# BOUNDARIES & STYLE
[numbered consolidated list]

# UNCLASSIFIED
List any responses that do not fit under the four headings.  
For each one, explain briefly why it does not fit.

# CONSOLIDATION NOTES
Provide short notes on:
- items that may have been over-merged
- items that are still ambiguous
- items that may need validation before interface design
- distinctions that should probably remain separate when creating future interface options
